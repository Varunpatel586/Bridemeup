## Project Overview

**Bridemeup** is a Delhi-focused, premium multi-tenant bridal beauty and luxury salon marketplace. The stack is split into a React/Vite/TypeScript frontend (TanStack Router + shadcn/ui) and a Python/FastAPI backend.

---

## Development Commands

### Unified (Recommended)

Start both frontend and backend concurrently from the **root** directory:

```bash
npm install   # installs `concurrently` in the root
npm run dev   # starts backend (localhost:8000) + frontend (localhost:8080)
```

> The backend is served by `uvicorn` (reload) and the frontend by Vite. The frontend port depends on availability (8080 or 5173).

### Frontend Only

```bash
cd frontend
npm install   # required after clone or dependency changes
npm run dev   # Vite dev server
npm run build # production build
npm run lint  # ESLint
npm run format# Prettier --write .
```

### Backend Only

```bash
cd backend
pip install -r requirements.txt   # Python deps; note mediapipe<0.10 for legacy solutions API
python -m uvicorn main:app --reload
```

Or via root script:

```bash
npm run backend
```

## Tech Stack & Architecture

### Frontend

| Layer | Technology | Notes |
|-------|------------|-------|
| Framework | React 19 + Vite | SPA build; TanStack Router for routing |
| Styling | Tailwind CSS 4 + shadcn/ui components | Component library in `components/ui/` вЂ” add via `npx shadcn add <component>` |
| Routing | TanStack Router | File-based routing in `src/routes/` |
| State | TanStack Query + React Context | Query client injected via router context |
| Auth | Supabase | `lib/AuthContext.tsx` wraps the app; redirects unauthenticated users to `/auth` with `?redirect=` param |
| AI Chat | Gemini (GenAI SDK) | Chatbot widget lives in `components/site/ChatbotWidget.tsx`, rendered globally in `__root.tsx` |

### Backend

| Layer | Technology | Notes |
|-------|------------|-------|
| Framework | FastAPI | Entry point: `backend/main.py` |
| Server | Uvicorn (reload) | Runs on port `8000` |
| AI | Google Gemini | `gemini_service.py` for face analysis and chat |
| Computer Vision | MediaPipe + OpenCV | Legacy `solutions` API requires `mediapipe<0.10`. See `face_analysis.py` for geometric face-shape classification |
| Auth | Supabase | JWT verification in `core/security.py` |

## Important Implementation Notes

### Frontend: TanStack Router + Vite Config

- The project uses **TanStack Router** with **file-based routing**. Routes live in `frontend/src/routes/<RouteName>.tsx`.
- Routes are generated automatically into `src/routeTree.gen.ts` вЂ” **do not edit manually**.
- The Vite config at `frontend/vite.config.ts` imports from **`@lovable.dev/vite-tanstack-config`**.
  - This config **already bundles** tanstackStart, viteReact, tailwindcss, tsConfigPaths, nitro, and others. Do **not** re-add them in `vite.config.ts` or the app will break from duplicate plugins.

### Frontend: Auth & Feature Gating

- Auth is handled via `AuthContext` (Supabase). Unauthenticated users are redirected to `/auth` with a `?redirect=` query param, then sent back after sign-in.
- The `/ai-analysis`, `/planner`, and `/auth` routes (plus the global chatbot widget) are gated behind authentication.

### Frontend: Backend API Calls

- The frontend calls the backend directly via `fetch` to `http://localhost:8000/api/...`. Ensure the backend is running for AI analysis and chat to function.
- Server functions (SSR) use `createServerFn` from `@tanstack/react-start`. Example in `frontend/src/lib/api/example.functions.ts`.

### Backend: MediaPipe Compatibility

- The backend uses `mediapipe<0.10` to retain the legacy `solutions` API. Avoid upgrading without checking for API breakage.
- Face-shape classification in `app/services/face_analysis.py` uses geometric ratios from MediaPipe face landmarks (468 points).

### Backend: API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/analysis/face` | Multipart image upload or base64 string; returns face shape + recommendation |
| `POST` | `/api/analysis/chat` | Chat request with history + face shape; returns Gemini-powered response |

### Environment Variables (Backend `.env`)

```env
GEMINI_API_KEY=your_key_here
RAZORPAY_KEY_ID=your_key_here
RAZORPAY_KEY_SECRET=your_key_here
SUPABASE_URL=your_supabase_url_here
SUPABASE_ANON_KEY=your_supabase_anon_key_here
SUPABASE_JWT_SECRET=your_supabase_jwt_secret_here
EMBEDDING_MODEL=all-MiniLM-L6-v2
```

- `GEMINI_API_KEY` is essential for AI analysis and chat endpoints.
- `SUPABASE_*` variables are required for auth and database operations.

### Frontend Styling Convention

- The app uses **extensive inline utility classes** directly in JSX (Tailwind + arbitrary values like `bg-[#FAFAFA]`).
- Colors follow a bridal theme: `#FAFAFA`, `#1A1A1A`, `#C5A880`. Avoid adding new arbitrary colors unless necessary вЂ” prefer Tailwind's built-in palette.
